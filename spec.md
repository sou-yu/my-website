# 横浜3日間旅行スマホ向けWebアプリ 仕様書

## 1. 目的

3日間の横浜旅行で、移動中にスマートフォンから素早く確認できる旅行支援用Webアプリを作成する。
本アプリは、1日ごとの行程、観光地情報、Googleマップによる移動経路、電車移動の詳細情報を、見やすく一元表示することを目的とする。

## 2. 想定利用シーン

- 旅行当日に現在の予定と次の移動先を確認する
- 観光地の写真を見て目的地を即座に認識する
- 公式サイトを開いて営業時間や休館日などの最新情報を確認する
- 電車の乗換、所要時間、利用路線をその場で確認する
- 1日の全体ルートを地図で把握する

## 3. 対象環境

- 利用端末: スマートフォンを最優先
- 想定画面幅: 360px - 430px
- 対応ブラウザ: iPhone Safari 最新系、Android Chrome 最新系
- 実装形態: HTML / CSS / JavaScript の静的Webアプリ

## 4. デザイン方針

- 明るめで清潔感のある配色とする
- 全体はシンプルに保ちつつ、余白、写真、カードUI、タイポグラフィで上質感を出す
- 旅行中の視認性を優先し、文字サイズは小さすぎないこと
- 操作を最小化し、スクロール中心で情報にアクセスできる構成とする
- 屋外利用を想定し、背景と文字のコントラストを十分に確保する

### デザインキーワード

- Bright
- Simple
- Elegant
- Travel-oriented
- Mobile-first

## 5. アプリ全体構成

アプリは1ページ構成を基本とし、上部タブまたは日別セクション切替で以下を表示する。

- Day 1
- Day 2
- Day 3
- 旅行全体の補助情報

### 推奨情報設計

1. ヘッダー
2. 3日分の切替タブ
3. 当日のサマリー
4. Googleマップ経路表示
5. タイムライン形式の行程
6. 観光地詳細カード
7. 電車移動詳細カード
8. 補助情報

## 6. 主要機能要件

### 6.1 日別プラン表示

- 3日分の旅行プランを日別に表示する
- 各日に対して以下を持つ
  - 日付
  - テーマまたはサブタイトル
  - 出発地点
  - 訪問スポット一覧
  - 移動手段
  - 1日の終了地点

### 6.2 Googleマップ埋め込み表示

- 各日ごとに、その日に回る主要地点の移動経路をGoogleマップ埋め込みで表示する
- 地図は `iframe` による埋め込みを前提とする
- ルート表示対象は神奈川県内の移動のみとする
- 出発地、経由地、到着地を日別に設定可能とする
- ユーザーは地図を見て1日の大まかな移動順を把握できること

### 地図表示ルール

- 神奈川県外を含む経路は埋め込み対象外とする
- 対象外の場合は、以下の代替表示を出す
  - 「この区間は地図埋め込み対象外です」
  - 出発地と目的地のテキスト情報
- 実装時は日別に埋め込みURLを保持する
- ルート表示に必要な地点名は表記ゆれを避け、正式名称で管理する

### 6.3 観光地カード表示

- 主要な観光地ごとにカードを表示する
- 各カードには以下を表示する
  - 写真
  - 観光地名
  - 短い説明
  - 住所またはエリア名
  - 滞在予定時間
  - 公式サイトへのリンク

### 観光地カード要件

- 写真は縦長スマホでも見やすい比率で表示する
- リンクは外部サイトとして開けること
- 公式サイトURLは視認しやすいリンクボタンで表示する
- 画像が取得できない場合はプレースホルダー画像を表示する
- 画像には代替テキストを設定する

### 6.4 電車移動の詳細表示

- 電車移動区間は、観光地情報とは別カードで詳細表示する
- 旅行中に見返しやすいよう、1移動ごとに独立したブロックとする

### 電車移動カードに含める項目

- 区間タイトル
- 出発駅
- 到着駅
- 利用路線名
- 乗車時間
- 乗換回数
- 乗換駅
- 出発予定時刻
- 到着予定時刻
- 補足メモ

### 電車移動カードの表示方針

- 「いま必要な情報」が上から順に見える構成とする
- 駅名と時刻は太字で強調する
- 乗換がある場合はステップ表示にする
- 徒歩移動が含まれる場合は、電車情報とは分けて表示する
- 遅延時に備え、補足メモ欄に代替案や注意事項を記載できるようにする

### 6.5 タイムライン表示

- 1日の行程を時系列で並べる
- 観光地と移動を交互に表示できるようにする
- ユーザーがスクロールするだけで次の行動を確認できること

### タイムライン要素

- 時刻
- 種別
  - Spot
  - Train
  - Walk
  - Meal
- タイトル
- 補足情報

### 6.6 実用補助情報

- 各日ごとに以下の補助情報を表示できるようにする
  - その日の持ち物メモ
  - 営業時間注意
  - 雨天時メモ
  - チケットや予約の注意事項
- 旅行全体で共通の注意事項も別セクションに掲載する

## 9. コンテンツ要件

### 必須コンテンツ

- 3日分の旅程
- 各日のGoogleマップ経路
- 主要観光地の写真
- 各主要観光地の公式サイトリンク
- 電車移動の詳細情報
- 旅行中の注意メモ

### コンテンツ管理ルール

- 観光地名、駅名、路線名は手入力ゆれを防ぐ
- 公式サイトURLはリンク切れ確認を行う
- 画像は表示速度を考慮して圧縮したものを使う
- 画像の権利関係が明確な素材のみ使用する
- 営業時間や運休など変動情報は、詳細本文に固定記載しすぎず公式サイト確認へ誘導する

## 10. 非機能要件

### 10.1 操作性

- 初見でも迷わず使えること
- スマホで片手操作しやすいこと
- タップ対象は十分な大きさを確保すること

### 10.2 表示性能

- 初期表示は3秒以内を目標とする
- 画像は遅延読み込みを検討する
- 1日ごとの表示切替は滑らかであること

### 10.3 可読性

- 日中屋外でも読める配色にする
- 重要情報はカード先頭に配置する
- 文字の最小サイズは可読性を損なわないこと

### 10.4 アクセシビリティ

- 画像に `alt` を付与する
- ボタンやリンクは識別しやすい見た目にする
- 色だけに依存しない情報伝達を行う

## 11. 実装方針

### 11.1 フロントエンド構成

- `index.html`
  - ページ骨組み
- `styles.css`
  - モバイルファーストのスタイル
- `app.js`
  - 日別切替、データ描画、動的レンダリング

### 11.2 実装方式

- SPA 風の1ページ構成
- データ駆動で各日の内容を描画する
- 日別タブ押下で、対象データを再描画または表示切替する
- HTML内にハードコードしすぎず、配列データからUIを生成する

### 11.3 地図実装方針

- 1日ごとに1つの埋め込み地図を表示する
- Googleマップ埋め込みURLは日別データで管理する
- 神奈川県内の経路のみ埋め込み有効とする
- 対象外の区間はテキスト表示へフォールバックする

## 13. 推奨スタイル仕様

### 配色

- ベース: 白またはごく薄いアイボリー
- アクセント: 水色、明るい青、やわらかいオレンジなど
- 文字色: 濃いグレー系

### 余白

- セクション間は広めに確保する
- カード内余白を十分に取り、窮屈さを避ける

### 角丸と影

- カードは中程度の角丸
- 影は弱めで軽やかにする

### タイポグラフィ

- 見出しは視認性の高いフォントウェイト
- 本文は読み疲れしにくいサイズと行間

## 14. 画面遷移方針

- 基本は単一ページ内遷移
- タブ切替以外の複雑な画面遷移は持たせない
- 公式サイトリンクのみ外部遷移とする

## 15. エラー・例外時の扱い

- 地図URLが無効な場合は地図の代わりに案内文を表示する
- 画像がない場合はプレースホルダーを表示する
- 公式サイトURLが未設定の場合はリンクボタンを非表示にする
- 電車時刻が未確定の場合は `未設定` と表示する

## 16. 実装時の注意点

- Googleマップ埋め込みは、旅行前に必ず全日分の表示確認を行う
- 実際の電車時刻はダイヤ改正の影響を受けるため、固定情報として過信させない
- 公式サイトでの最終確認を促す導線を残す
- 旅行中は通信状況が不安定な可能性があるため、本文情報は可能な限りアプリ内にも保持する
